**命令行终端**

杀死相同的东西

```shell
taskkill /f /im cmd.exe
```

- `/f`：`force`，强制
- `/im`：俗称映像名称，`image`

## 服务

进程完了，这个就好理解了。设想在Linux环境下，全是黑屏，想要打开一个app，需要在终端敲命令，Windows虽然预装有`桌面环境`这么个东西，但是还是离不开终端的，所以我们假设想要开机就启动`explorer.exe`，手动方法肯定如下

**打开终端，输入explorer命令运行**

![image-20260324215219832](/posts/系统基础笔记/windows进程与服务/assets/image-20260324215219832.png)

所以这么个操作想让操作系统自己完成，就需要让系统知道，那么就可以写一个服务的方式去做到。不过Linux是一个`service`文件，Windows是存在**注册表**中的。不过似乎不用自己写服务，Linux是可以自己写的。

**图形界面**

Win+R - `services.msc`

![image-20260324215500012](/posts/系统基础笔记/windows进程与服务/assets/image-20260324215500012.png)

右键即可管理

![image-20260324215521670](/posts/系统基础笔记/windows进程与服务/assets/image-20260324215521670.png)

**命令行终端**

启动服务

```shell
net start [某个服务名字]
```

停止进程

```shell
net stop [某个服务名字]
```
